  <files>database/migrations/2026_04_28_000001_create_exams_table.php, database/migrations/2026_04_28_000002_create_exam_questions_table.php, app/Models/Exam.php, app/Models/ExamQuestion.php, app/Services/ExamService.php</files>
  <action>
    Create the database schema and models for the exam system:

    1. **Migration: Create exams table**
       - Use `php artisan make:migration create_exams_table`
       - Fields: `id` (bigIncrements), `user_id` (foreignId to users), `title` (string, nullable), `description` (text, nullable), `duration_minutes` (integer, default 60), `passing_score` (integer, default 50), `status` (enum: draft, published, archived, default 'draft'), `deleted_at` (softDeletes), `created_at`, `updated_at`
       - Index on `status` for filtering

    2. **Migration: Create exam_questions table**
       - Use `php artisan make:migration create_exam_questions_table`
       - Fields: `id`, `exam_id` (foreignId, cascade delete), `question_bank_question_id` (foreignId to question_bank_questions, cascade delete), `custom_marks` (integer, nullable - overrides question marks), `order` (integer, default 0), `created_at`, `updated_at`
       - Unique constraint on (`exam_id`, `question_bank_question_id`)
       - Index on `exam_id` and `order`

    3. **Model: Exam.php**
       - Use `php artisan make:model Exam`
       - Fillable: `user_id`, `title`, `description`, `duration_minutes`, `passing_score`, `status`
       - Casts method: `status` (enum using backing strings), `duration_minutes` (integer), `passing_score` (integer)
       - Relationships:
         - `creator(): BelongsTo` to User
         - `questions(): HasMany` to ExamQuestion (with orderBy('order'))
         - `examQuestions(): HasMany` to ExamQuestion (alias)
       - Scopes: `scopePublished($query)` where status = 'published'
       - Use SoftDeletes trait
       - Add `toExamShieldFormat()` method that returns array with exam data and questions

    4. **Model: ExamQuestion.php**
       - Use `php artisan make:model ExamQuestion`
       - Fillable: `exam_id`, `question_bank_question_id`, `custom_marks`, `order`
       - Casts method: `order` (integer), `custom_marks` (integer, nullable)
       - Relationships:
         - `exam(): BelongsTo` to Exam
         - `question(): BelongsTo` to QuestionBankQuestion
       - Accessor: `getEffectiveMarksAttribute()` returns custom_marks ?? question->marks

    5. **Service: ExamService.php**
       - Create in `app/Services/`
       - Methods:
         - `createExam(User $user, array $data): Exam` - creates exam with user_id
         - `updateExam(Exam $exam, array $data): bool` - updates exam fields
         - `toggleStatus(Exam $exam): string` - toggles between draft/published/archived
         - `deleteExam(Exam $exam): bool` - soft deletes
         - `getExamsWithStats(): Collection` - returns exams with question count and attempt count
       - Follow pattern from CtfService.php (constructor injection if needed)

    Run `vendor/bin/pint --dirty` after creating files.
  </action>

       - Properties: `$search` (string), `$statusFilter` (nullable string), `$perPage` (integer, default 20)

       - Methods: `delete($id)` (soft delete with trashed check), `toggleStatus($id)` (calls service)
       - Render: returns view with exams paginated
       - Follow pattern from `@app/Livewire/QuestionBank/Index.php`

    2. **Livewire: Create.php** (Exam creation)
       - Use `php artisan make:livewire Admin/Exams/Create`
       - Properties: `$title`, `$description`, `$duration_minutes` (default 60), `$passing_score` (default 50)
       - `rules()`: title required string, description nullable string, duration_minutes required integer min:1, passing_score required integer min:0 max:100
       - `save()` method: validates, calls ExamService::createExam(auth()->user(), data), redirects to index with success
       - `saveAndContinue()` method: saves and resets form for another (optional)
       - Render: returns create form view

    3. **Livewire: Edit.php** (Exam editing)
       - Use `php artisan make:livewire Admin/Exams/Edit`
       - `mount(Exam $exam)`: sets properties from $exam
       - Properties same as Create, plus `$exam` (the model)
       - `save()` method: validates, calls ExamService::updateExam($this->exam, data), redirects with success
       - Render: returns edit form view
